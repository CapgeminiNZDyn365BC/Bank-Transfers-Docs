# Release Notes

| Version  | Date        | Notes |
|----------|-------------|-------|
| 27.5.8.0 | 15-JUL-2026 | Added an option to Capgemini Bank Transfers Setup that will prevent payee details from being copied to a new document when a one time vendor document is copied. |
| 27.0.6.0 | 28-JAN-2026 | Fix number allocation on suggest payments issue. |
| 27.0.0.0 | 14-OCT-2025 | Make compatible with version 27. |
| 26.0.0.0 | 11-FEB-2025 | Fix issue with ASB MT9 file format. Allow Suggest Payments report to have report options saved and copied. |
| 25.0.2.0 | 30-OCT-2024 | Delete payment errors before trying to create a new export. Apply fix in 24.0.1.0 to Australian EFT export. |
| 25.0.1.0 | 24-SEP-2024 | Extend Vendor Payment Buffer instead of Payment Buffer |
| 25.0.0.0 | 11-SEP-2024 | Remove obsolete code and make compatible with version 25. Add new events to EFT Transaction Create Transaction Lines process to allow additional validation to be performed before the bank file is downloaded. |
| 24.0.1.0 | 16-JUL-2024 | Fix issue where an error in the payment batch does not get reported when the last line in the batch does not have errors and different document numbers are used on each line. |
| 24.0.0.0 | 02-APR-2024 | Update to work with release 2024 wave 1. Allow description to be edited on payment journal line for the balancing bank line only after the file has been exported. Address validation removed from Payee address fields as the previous routines were moved into an inaccessible library in this version. |
| 23.0.0.0 | 09-SEP-2023 | Rebranded as Capgemini Bank Transfers. Added options to determine where payee name should come from within the payment file. Standard options are Customer Name, Customer Bank Name for customers and Vendor Name, Vendor Bank Name for Vendors. There are also events that allow overriding the standard options. Use the Capgemini Bank Transfers Setup screen to set the preferred source of payee name. The default values for the new fields match the old behaviour from prior to this update. |
| 20.0.5.0 | 15-JUN-2022 | Add new setup for "Hash Totals to Show" which will allow combinations of NZ Hash Total and Westpac NZ Hash Total to be selected on the Intergen Bank Transfers Setup page. Removed error check when creating an EFT payment that prevented the user from applying more to an open entry than the remaining amount as this meant early settlement discounts could not be used. |
| 20.0.4.0 | 12-MAY-2022 | Fix issue where a user sends an email and doesn't have permission to the EFT Emails. |
| 20.0.3.0 | 02-MAY-2022 | Fix issues where permission set migration causes upgrades to fail in some circumstances. |
| 20.0.2.0 | 05-APR-2022 | Add code to update old 'BANK TRANSFERS' permission set to use the new 'IIIBNK ALL' permission set in its place. |
| 20.0.1.0 | 01-APR-2022 | Add permissions to read and modify emails in the outbox and sent emails. App permissions modified to use new object with code 'IIIBNK ALL' and caption of 'Bank Transfers'. |
| 20.0.0.0 | 24-MAR-2022 | Update to work with release 2022 wave 1. Improve integration with Email accounts to show status of EFT Emails and provide a link between the EFT Email and the Outbox or Sent items. Add customer details to the payment breakdown report. Add a new check to ensure that the first line of the payment journal batch has a value in Document No. as this will be used as the EFT Transaction No. |
| 18.3.1.0 | 23-AUG-2021 | Update to work with release 2021 wave 1. Changed version numbering to match the Major.Minor of the target Base Application. Added External Document No. and Message to Recipient as reference rules. Allow dimensions to be edited on payment journal lines after the file has been exported. Truncate reference fields to the correct length in all export file types. Include ABN and GLN (NZBN) from Company Information in the dataset for the remittance report. |
| 1.0.9.1 | 24-MAR-2021 | Keep dimensions on payment journal lines after generating a payment file. Fix error with Payment Breakdown report when there is more than one Payment type Gen. Journal Template. Add setup of new options to OnUpdate trigger. |
| 1.0.9.0 | 01-FEB-2021 | It is now possible to override reference rules for individual vendor bank account, customer bank account, and one-time vendor payee details. Added support for new email module with Bank Transfers email scenario. Multiple recipients supported in email address. New email subject template in setup. Added tooltips throughout the application. Added EFT Email and EFT Transaction as tables in retention policies. Added applied-to description to remittance report dataset. Fix error where ABN No. and Registered fields were not copied through to the posted invoice for one-time vendor payments. Updated to support 2020 Wave 2. Added new developer events for overriding remittance report as an extension. Fixed error in suggest payments that applied entries without adding the journal line. Obsolete field in 50000..99999 range and move data to new field to avoid AppSource rejection rule. |
| 1.0.8.0 | 01-APR-2020 | All reference rules are now selectable regardless of the reference name they are being used for. Fix missing new line in Kiwibank GIFTS file format. Fix missing total amount in ASB MT9 file format. Added support for new Subscription Service. Updated to support 2020 Wave 1. |
| 1.0.6.0 | 1-OCT-2019 | Added support for Kiwibank GIFTS File Format as an option for Bank Accounts that begin with '38'. Allow a Word Layout for the Remittance Advice report to be configured as the Email body for the emailing of the remittance. Layout is specified on the Intergen Bank Transfers Setup. Allow a different email address to be shown on the Remittance Advice report than the email on the Company Information card. New setting to override is on the Intergen Bank Transfers Setup. Update to support 2019 Wave 2.Increased field sizes to allow for changes in standard application field sizes. Correct SaaS licensing to use the Azure Active Directory Tenant ID and not the Tenant ID as the client-identifying part of the license check. Added support for ABN fields to One-time vendor. Added new integration events to improve extensibility. |
| 1.0.0.2 | 1-JUL-2019 | The first release available in AppSource. |
